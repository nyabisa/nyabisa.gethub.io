Undestanding Basic Web Layout

Starting With <html>

We'll start form the very beginning.
At the very top of every coding sheet you will see the tag <!Doctype html>
This lets the computer know that you are going to be using html. The computer needs to know what language you will be writing in so it can understand you.
HTML stands for HyperText Markup Language.

Following the Doctype tag there will be an <html> tag. Every tag needs to be opened and closed. This means that they will have an opening tag which in this case is <html> and the closing tag will be </html>. The only difference is the slash at the beginning. This will let the computer know where your code begins and ends

Back to short cuts

Moving on to the <head> tag

The <head> tag is necessary for the information that will be used in the web page. It is referred to as metadata. Inside this tag will be the <title></title> which is required to create a page.
For future reference, when you place information "inside" a tag you are placing it between the opening and closing tag.

Inside the head tag you will also find this very specific and particular tag: <meta charset="utf-8">.
This tag lets the computer know what format to use to turn the code into readable characters for wahtever you're making.

Here's what it would look like so far:

<!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

</html>

I know it looks confusing but hopefully it's a little bit less scary now that you have an idea what they all do.
But remember, this is barely scratching the surface. So keep on reading if you want to gain even more knowledge on how programming works

Back to short cuts

Let's talk <body>

Inside of the <body> tag you will find absolutely everything that has to do with the content of a web-page. You will need to use other tags inside the body tag to make headings, paragraphs, bullet points, numbered lists, insert images, ect., ect.

We'll talk about the headings and paragraphs first.
To make a heading you use the <h> tag. This tag will always be accompanied by a number inside it. There's 6 headings. Number 1 is the largest, so it's used for the most important headings. As the numbers go down to 6, they are used for less important headings. For example the heading tag I used above was <h4>. In code, it looks like this:

<h4> Whatever you want to write </h4>

To make a paragraph, or any type of text body that isn't a heading, you will use the <p> tag. It's really quite simple. It looks like this:

<p> Once again whatever you want to write</p>

Back to short cuts

Breaks

When you open a <p> tag the computer will read it as a paragraph without end. I can keep on writing and writing and the computer will keep reading it as a whole paragraph. What if I want a space between sentences? In a word page you can just click enter and it takes you right underneath your last sentence. In code you need to use breaks.
Breaks allow you to
go underneath a sentence
Inside of a <p> tag
whenever you want.
I think I made my point. The break tag is <br>. It's used at the end of whatever sentence you want to space out
It look like this:

<p> I'm writing a sentence but now I want to <br>
break it up and bring it underneath </p>

It can also be used to add additional spaces to make several paragraphs within a <p> tag. That will look like this:

<p> i'm writing and writing <br>
<br>
And now I'm writing again <p>

That <br> tag in the middle that's alone will be registered as an empty space.

Back to short cuts

Bullet Points

I wanted to point out the table of contents I have at the very top. You can see that I have bullet points. How did I do that? I used the <ul> tag "ul" stands for "unordered list". The unordered list gives you bullets point. Inside of the <ul> tags you're going to have your bullet point. For them to actually show you need to let them computer know that you want write things inside the list You're going to do this by using the tag <li> </li>. "li" stands for list item. It looks like this:
<ul> <li > <a href="#html">Starting with html</a></li> </ul>
There's some extra stuff going on in that tag but I'll go over that later on in "Jumping Around"

What about Pictures?

Say you want to add pictures to whatever webpage you're making. How exactly do you do that? You use a tag.

<img src="">

You might be able to guess that "img" stands for image. "Src" stands for source.
When you have an equal sign after a word we call this an attribute. An attribute will be followed by an equal sign and quotation marks. Inside the quotation marks there will be a declaration.
An attribute tells a computer what to do and a declaration tells the computer how to do it.
The declaration of the attribute src="" will be the URL of the pciture you're selecting.

Back to short cuts

How to emphasize words

To emphasize words we either italicize them or we make them bold.
How do we do this? If you said tags, you're right!
To italicize a word we use the <em> </em> tag. "em" stands for emphasize. Pretty straight forward.

To make a word bold you use the <strong> </strong> tag. I believe this is pretty self-explanatory.

Back to short cuts

Styling (CSS)

Now that we've gone over how to put text on a webpage, let's go over on how to style it. First we'll learn how to change the colors of the text and the background.
CSS stands for Cascating Style Sheet. CSS is a completely different language than HTML. We'll be using CSS to style so we'll need to open a <style> </style> tag. This tag will let them computer know that anything inside it will be using CSS. This tag will be inside of the head tag. It'll look like this:

<!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

-----<style> </style>

</html>

Back to short cuts

Selectors

To be able to change the color of a text you need to select what portion you're going to be changing first. You do this by using a selector. They're pretty easy and straight-forward to use.
If you want to select a paragraph you'll use the selctor "p". Selectors are followed by {squiggly paranthesis}. You need to make sure that those are the ones you use otherwise the selctors won't work.
Inside the paranthesis you will use the declaration color:; to change the color of a text.
It'll look like this:
 <!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

-----<style>

----------p {
---------------color: rgb(0, 0, 0);
----------}

-----</style>

</html>

Now all of your <p> tags have the same color with just one selector. The computer recognizes colors through numbers. You could always use words for simple colors such as blue, red, green, ect. The word would go inbetween the colon and semi-colon with no paranthesis.
So it would like this:

<style> p{color: blue;} </style>

To get a more specific color you use the rgb color picker, which looks like the code above with the numbers in it. By inputing rgb( ), you will immediately see a little square and can then pick whatever color you want. I personally prefer the color picker because I find it makes the webpage more interesting.

Back to short cuts

ID's

The only downside to a selector is that it's generalized. If you choose a "p" tag it'll color all the "p" tags. What if you want to change the color of a specific paragraph? That's where the ID comes in.
By giving a tag a unique ID you can use it to style a specific section. Thats sounds confusing but it's actually quite simple.
Let's continue with the <p> tag. Let's say I have five different <p> tags but I want to color the first one purple. The first thing I need to do is give the tag an ID. It'll look like this:

<p id="purple"> Whatever you wrote </p>

I chose to give the tag the id "purple". Now to change the color I need to go to the styling tag. To select an ID you need to use a # sign. This lets the computer know to look for the ID that you're about to write.

<!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

-----<style>

#purple {
----- color: purple;
}
-----</style>

</html>

Now you have the id inside the style tag and you have changed the color to purple. Only the paragraph that has that ID will change color. You also need to keep in mind that ID's are case sensititve. They both need to be written exactly the same way.

Back to short cuts

Classes

Now let's say that I had a three different headings, and a paragraph and I wanted them all to be the color blue
I could always give them all seperate IDs and then select the IDs and give them color delcarations, however that's a lot of work. In this case we can use a class. Just like the ID the class goes inside the opening tag, it has to be a unique name, and it's case-sensitive. It would look soemthing like this:

<h1 class="blue"> This is my first heading </h1>
<h2 class="blue"> This is my second heading </h2>
<h3 class="blue"> This is my third heading </h3>
<p class="blue"> This is my paragraph <p>
Now that they all have the same class you can go to your styling tag and put the class in there and change the color to blue.
When you're selecting a class you have to put a period in front of the class so the computer know that it has to look for a class.
That will look like this:

<!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

-----<style>

.blue {
----- color: blue;
}
-----</style>

</html>

Now everything that has the class "blue" is the color blue.

Back to short cuts

Background Color

Now our text looks pretty but the background is still plain and boring. Don't worry we can change that. If we want to select the whole entire page all you need to do is go to the style tag and select the <body> tag.
Like I said before, everything that will be seen in the webpage will be in the body tag. This includes the background.
The code will look like this:
<!Doctype html>
<html>

-----<title> "Title" </title>
-----<meta charset="utf-8">

-----<style>

----------body {
---------------background-color: rgb(0, 0, 0);
}

</html>

If you're trying this out on a code sheet you may have noticed that the tags and the color declarations turn blue, while the IDs and classes turn green. This is because the computers have a color-coded system to make it easier for us to read.
Instead of everything being black, everything has a different color so you can easily spot and differentiate between code.

Links

What if you want to put a link into a webpage? You guys know what a link is, right? You click on a text and it takes you to a whole new page. Well we're going to learn how to do this.
First we're going to need to learn a new tag. <a></a>. This is the attachment tag. Guess what it does. That's right, it attaches one thing to another.
Inside the first tag we're going to need to put a reference so the computer knows where to go. This is called href="". You might be thinking this already but href is an attribute.
Href stand for HyperReference. This is going to be your link, in this case the URL. Inside the "a" tags you''ll have the text that you'll want people to click on. It'll look like this:

<a href="link"> Click here </a>

The text should come out looking something like this:

Click here

If you clicked on it, it probably took you to the page I'm currently working on right now.
The reason why the link is the same color as the rest of the text and not blue like the "back to short-cuts" link I have is because I have all <a> tags under a selector that has a color declaration, pretty cool, right?

Jumping Around a Web-Page

Now I want to be able to make a link inside of a web-page that will allow be to jump around without having to scroll all the way up or down. We do this by using the same tag as above
However, instead of a URL code in the href="" we'll put in an ID. Let's take this very webpage as an example. Above I have the heading "Jumping Around a Web-Page".
If you go all the way up you'll see that I have a table a contents with a bullet-point and if you click on it it'll take you right back down here. Since you already know how to use the <a> tag I'll just show you what the code looks like

<h4 id="jump-around" class="text-color">Jumping Around a Web-Page</h4>

Inside the <h4> tag I have an ID named "jump-around". This is the ID I used to link this text to the bullet point all the way at the top. Let's see what that looks like

<li> <a href="#jump-around">Jumping around a Web-Page </a> </li>

As you can see I used the ID "#jump-around" for the href and wrote what I wanted you guys to see when clicking on the link
